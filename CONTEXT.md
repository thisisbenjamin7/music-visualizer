# Music Visualizer

Routes tasks to the correct stage for building and deploying a music visualizer web app.

## Task Routing

| Task Type | Go To | Description |
|-----------|-------|-------------|
| Define concept | `stages/01-concept/CONTEXT.md` | Choose platform, visualizer style, OS template, and color scheme |
| Scaffold project | `stages/02-initiation/CONTEXT.md` | Set up project files and API credentials |
| Build app | `stages/03-execution/CONTEXT.md` | Write the visualizer code |
| Test app | `stages/04-testing/CONTEXT.md` | Verify API integration and audio sync |
| Deploy | `stages/05-publishing/CONTEXT.md` | Publish to a public domain |

## Shared Resources

| Resource | Location | Contains |
|----------|----------|----------|
| App identity | `shared/app-identity.md` | App name, domain, audience |
| Platform specs | `shared/platform-specs.md` | Music platform API endpoints and auth patterns |
